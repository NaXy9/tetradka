# Тетрадка · Tetradka

> Маркетплейс репетиторов со встроенными видеозанятиями и AI-конспектами после каждого урока.
> *Tutoring marketplace with built-in video lessons and AI-generated lesson notes.*

**Занятие, которое остаётся с тобой.** Ученик находит репетитора, бронирует слот и занимается
прямо в приложении — без Zoom и пересылки ссылок. Через несколько минут после урока он получает
запись и конспект: главные мысли с таймкодами, термины и домашнее задание. Ничего не теряется.

## Как это работает

1. **Бронирование.** Каталог репетиторов с видео-визитками и отзывами из реальных занятий.
   Слоты показываются в часовом поясе ученика; оплата удерживается в холде до конца урока.
2. **Занятие.** Видеозвонок на платформе (LiveKit/WebRTC): чат, демонстрация экрана,
   серверная запись с согласия обеих сторон.
3. **Конспект.** Запись автоматически проходит пайплайн Whisper → LLM, и ученик получает
   структурированный конспект: тезисы с кликабельными таймкодами, ключевые термины и домашку.

## Архитектура

```
┌────────────────────────────┐          ┌──────────────────┐
│  React Native (Expo)       │─WebRTC──▶│   LiveKit SFU    │
│  приложение ученика        │          │  (видеокомнаты,  │
│  и репетитора              │          │   TURN, чат)     │
└─────────────┬──────────────┘          └────────┬─────────┘
              │ REST + JWT                       │ room composite
              ▼                                  ▼
┌────────────────────────────┐          ┌──────────────────┐
│  Django + DRF              │◀─webhook─│  LiveKit Egress  │
│  users / booking / payments│          │  (запись → MP4)  │
│  выдача токенов комнат     │          └────────┬─────────┘
└─────────────┬──────────────┘                   │ MP4
              │ enqueue                          ▼
              ▼                          ┌──────────────────┐
┌────────────────────────────┐           │   S3 / MinIO     │
│  Celery + Redis            │◀──файл────│ записи, конспекты│
│  напоминания, очистка,     │           └──────────────────┘
│  AI-пайплайн:              │
│  Whisper → транскрипт →    │
│  LLM → конспект + ДЗ       │
└────────────────────────────┘
```

Технические решения, которыми проект гордится:

- **Запись без бота.** Занятие пишется на серверной стороне через LiveKit Egress —
  composite-запись комнаты уходит в S3 и триггерит AI-пайплайн по вебхуку.
- **Конкурентно-безопасное бронирование.** Транзакция + `select_for_update` в приложении,
  exclusion constraint (`tstzrange` + GiST) в PostgreSQL как последняя линия обороны.
  Два одновременных запроса на один слот → ровно одна бронь, всегда.
- **Честные таймзоны.** Всё время в БД — UTC; правила доступности репетитора хранятся в его
  поясе и разворачиваются в слоты на лету. Переходы на летнее время покрыты тестами.
- **Провайдеры за абстракциями.** Транскрипция, LLM и платежи — сменные реализации
  (`TranscriptionProvider`, `SummaryProvider`, `PaymentProvider`) с mock'ами для тестов.
- **Идемпотентные вебхуки** (LiveKit, платежи) с проверкой подписи; Celery-задачи с retry
  и видимыми статусами пайплайна.
- **Готовность к масштабированию.** Именованные очереди Celery (`default` / `media` / `gpu`),
  S3 API с первого дня, все адреса из env — вынос видео или обработки на отдельный сервер
  не требует изменений в коде.

## Быстрый старт

```bash
git clone https://github.com/NaXy9/tetradka.git && cd tetradka
cp .env.example .env
docker compose up -d          # postgres, redis, minio, livekit, egress
make bootstrap                # миграции, суперпользователь, seed-данные (10 репетиторов)
```

API поднимется на `http://localhost:8000` (Swagger — `/api/schema/swagger-ui/`),
MinIO-консоль — на `:9001`. Мобильное приложение:

```bash
cd mobile && npm install && npx expo start
```

Для работы AI-пайплайна укажите в `.env` ключ провайдера транскрипции — без ключа
пайплайн работает на mock-провайдере и возвращает демонстрационный конспект.

## Тесты

```bash
pytest -x -q                  # юнит- и интеграционные тесты
ruff check . && black --check .
```

Обязательные зоны покрытия: пересечение слотов при конкурентных запросах, таймзоны и DST,
статусная машина брони, идемпотентность вебхуков, расчёт комиссии.

## Статус разработки

- [ ] **Этап 1 — ядро бронирования:** auth, профили, каталог, слоты, конкурентно-безопасная бронь
- [ ] **Этап 2 — видео:** LiveKit, комнаты занятий, чат, демонстрация экрана (web)
- [ ] **Этап 3 — записи и AI-конспекты:** Egress → S3 → Whisper → LLM → пуш ученику
- [ ] **Этап 4 — платежи:** холд при бронировании, комиссия, возвраты, баланс репетитора
- [ ] **Этап 5 — полировка:** отзывы, лендинг, веб-версия, деплой демо

## История проекта

Тетрадка выросла из моего дипломного проекта [RecAcad](https://github.com/NaXy9/RecAcad) —
сервиса, где headless-бот заходил в чужие конференции, записывал их и генерировал конспекты.
Здесь та же идея доведена до продукта, а архитектура стала чище: конференция теперь своя,
и запись — просто свойство платформы, а не трюк с ботом.
